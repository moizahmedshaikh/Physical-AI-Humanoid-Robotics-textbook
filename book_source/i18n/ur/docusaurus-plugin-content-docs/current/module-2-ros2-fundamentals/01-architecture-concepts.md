# ROS 2 آرکیٹیکچر اور بنیادی تصورات

## تعلّم کے اہداف
*   ROS 2 کی بنیادی آرکیٹیکچر اور اس کی تقسیم شدہ نوعیت کی وضاحت کریں۔
*   ROS 2 کے بنیادی تصورات جیسے کمپیوٹیشنل گراف اور DDS کی شناخت کریں اور ان کی وضاحت کریں۔
*   ROS 1 اور ROS 2 کے کلیدی ڈیزائن کے انتخاب اور بہتریوں کے درمیان تمیز کریں۔
*   تعامل اور انٹرو ایکشن کے لیے `ros2` کمانڈ لائن ٹولز کا مقصد سمجھیں۔
*   بنیادی ROS 2 ورک سپیس سیٹ اپ کریں اور ایک سادہ پیکج تخلیق کریں۔

## جامع مواد

روبوٹ آپریٹنگ سسٹم (ROS) روبوٹکس سافٹ ویئر ڈیولپمنٹ کے لیے ایک حقیقی معیار بن گیا ہے۔ ROS 2 ROS کی اگلی نسل ہے، جسے جدید روبوٹکس ایپلی کیشنز کی ضروریات کو پورا کرنے کے لیے دوبارہ ڈیزائن کیا گیا ہے، بشمول بہتر ریل ٹائم صلاحیتیں، سیکورٹی، اور وسیع پیمانے پر پلیٹ فارم کی حمایت۔ یہ ایک لچکدار فریم ورک فراہم کرتا ہے جو سافٹ ویئر اجزاء کی ماڈولریٹی اور دوبارہ استعمال کے فروغ کے ذریعے پیچیدہ روبوٹ کے رویوں کی ترقی کو فعال کرتا ہے۔

### ROS 2 کیا ہے؟
ROS 2 سافٹ ویئر لائبریریز اور ٹولز کا ایک سیٹ ہے جو آپ کو روبوٹ ایپلی کیشنز تیار کرنے میں مدد دیتا ہے۔ ڈرائیورز سے لے کر جدید ترین الگورتھم تک، اور طاقتور ڈیولپر ٹولز کے ساتھ، ROS آپ کے اگلے روبوٹکس پروجیکٹ کے لیے ہر چیز فراہم کرتا ہے۔ ROS 2 ROS 1 کی کامیابیوں پر تعمیر کرتا ہے، لیکن ایک بنیادی طور پر دوبارہ تعمیر شدہ کور کے ساتھ جو ریل ٹائم کارکردگی، متعدد روبوٹ سسٹم، اور سیکورٹی جیسے علاقوں میں محدودیت کو حل کرتا ہے۔

### ROS 2 کے کلیدی ڈیزائن اہداف
*   **متعدد RTOS (ریل ٹائم آپریٹنگ سسٹم) کی حمایت**: صنعتی اور حفاظتی تنقیدی ایپلی کیشنز کے لیے ضروری قطعی رویے کو فعال کرتا ہے۔
*   **متعدد روبوٹ سسٹم**: روبوٹس کی فلوٹس کو آسانی سے اتارنے اور انتظام کرنے کے لیے ڈیزائن کیا گیا ہے۔
*   **بہتر سیکورٹی**: تصدیق، خفیہ کاری، اور رسائی کے کنٹرول کے لیے میکنزم فراہم کرتا ہے۔
*   **سروس کا معیار (QoS)**: ڈیولپرز کو مواصلت کی قابل اعتمادی، تاخیر، اور گنجائش کو ترتیب دینے کی اجازت دیتا ہے۔
*   **تقسیم شدہ نوعیت**: ڈیٹا ڈسٹری بیوشن سروس (DDS) کا فائدہ اٹھاتے ہوئے مرکزی ماسٹر کے بغیر ہم مرتبہ سے ہم مرتبہ مواصلت کے لیے۔

### ROS 2 بنیادی تصورات

#### 1. کمپیوٹیشنل گراف
ROS 2 میں کمپیوٹیشنل گراف ڈیٹا کو پروسیس کرنے والے ROS 2 عناصر کے نیٹ ورک کو اشارہ کرتا ہے۔ اس گراف میں شامل ہیں:
*   **نوڈس**: ایسے ایگزیکوٹیبل جو کمپیوٹیشن کرتے ہیں۔ ایک روبوٹ سسٹم میں عام طور پر بہت سے نوڈس ہوتے ہیں، جہاں ہر ایک مخصوص کام کے ذمہ دار ہوتا ہے (جیسے، ایک موتور کو کنٹرول کرنا، کیمرہ ڈیٹا کو پروسیس کرنا، راستہ منصوبہ بندی کرنا)۔
*   **ٹاپکس**: نامزد بسز جن کے ذریعے نوڈس پیغامات کا تبادلہ کرتے ہیں۔ ٹاپکس ایک شائع کریں/سبسکرائب کمیونیکیشن ماڈل ہیں، جہاں ایک نوڈ کسی ٹاپک پر پیغامات شائع کرتا ہے، اور دیگر نوڈس اس ٹاپک سے پیغامات وصول کرنے کے لیے سبسکرائب کرتے ہیں۔
*   **سروسز**: ایک درخواست/جواب کمیونیکیشن ماڈل۔ ایک سروس کلائنٹ نوڈ کو سروس سرور نوڈ کو ایک درخواست بھیجنے اور جواب وصول کرنے کی اجازت دیتی ہے۔ یہ عام طور پر ہم وقت سروس، بلاکنگ آپریشنز کے لیے استعمال کیا جاتا ہے۔
*   **ایکشنز**: ایک طویل وقت تک جاری رہنے والی درخواست/فیڈ بیک/نتیجہ کمیونیکیشن ماڈل۔ ایکشنز ان کاموں کے لیے مناسب ہیں جن میں مکمل ہونے میں طویل وقت لگتا ہے اور جن کے لیے کلائنٹ کو ممکنہ طور پر مسلسل فیڈ بیک چاہیے اور درخواست منسوخ کرنے کی صلاحیت ہو۔ وہ ٹاپکس اور سروسز پر مبنی ہیں۔
*   **پیرامیٹرز**: نوڈس کے لیے کنفیگریشن ویلیوز۔ نوڈس پیرامیٹرز کو ظاہر کر سکتے ہیں جن کو رن ٹائم پر تبدیل کیا جا سکتا ہے یا فائلوں سے لوڈ کیا جا سکتا ہے۔

#### 2. ڈیٹا ڈسٹری بیوشن سروس (DDS)
ROS 2 کے مواصلاتی لیئر کے دل میں ڈیٹا ڈسٹری بیوشن سروس (DDS) ہے۔ DDS تقسیم شدہ نظاموں میں ریل ٹائم، اسکیل ایبل، اور ہائی کارکردگی ڈیٹا ایکسچینج کے لیے ایک بین الاقوامی معیار ہے۔ ROS 2 میں DDS کی کلیدی خصوصیات میں شامل ہیں:
*   **مرکزی داری سے پاک مواصلت**: ROS 1 کے ماسٹر-سلیو آرکیٹیکچر کے برعکس، DDS نوڈس کے درمیان براہ راست ہم مرتبہ سے ہم مرتبہ مواصلت کو فعال کرتا ہے۔ یہ مضبوطی اور اسکیل ایبلٹی کو بڑھاتا ہے۔
*   **ڈسکوری**: نوڈس خود بخود ایک دوسرے کو دریافت کرتے ہیں بغیر مرکزی سرور کی ضرورت کے۔
*   **سروس کا معیار (QoS)**: DDS ڈیولپرز کو مواصلت کی خصوصیات کو باریک کرنے کے قابل بنانے کے لیے QoS پالیسیز کا ایک غنی سیٹ فراہم کرتا ہے (جیسے، قابل اعتمادی، تاریخ، ڈیڈ لائن، زندگی) تاکہ اطلاقیہ کی ضروریات کو میل کیا جا سکے۔
*   **متبادلیت**: مختلف DDS ایمپلیمنٹیشنز ایک دوسرے کے ساتھ بات چیت کر سکتی ہیں، جو زیادہ لچک کی اجازت دیتا ہے۔

### ROS 1 بمقابلہ ROS 2: کلیدی فرقوں

| خصوصیت | ROS 1 | ROS 2 |
| :---------------- | :-------------------------------------- | :----------------------------------------------- |
| **مواصلت** | ROS ماسٹر (مرکزی سرور) | DDS (غیر مرکزی، ہم مرتبہ سے ہم مرتبہ) |
| **ریل ٹائم** | محدود، ریل ٹائم کے لیے ڈیزائن نہیں کیا گیا | بہتر ریل ٹائم صلاحیتیں، QoS |
| **متعدد روبوٹ** | چیلنجنگ، نام کے تنازعات | متعدد روبوٹ سسٹم کے لیے ڈیزائن کیا گیا |
| **سیکورٹی** | کم از کم باہر کے ڈبے سے | انضمام شدہ سیکورٹی (تصدیق، خفیہ کاری) |
| **سپورٹڈ OS** | لینکس (زیادہ تر) | لینکس، ونڈوز، میک او ایس، RTOS |
| **API زبان** | C++, Python (rospy, roscpp) | C++, Python (rclpy, rclcpp) |

### `ros2` کمانڈ لائن ٹولز
ROS 2 ایک چلنے والے ROS 2 سسٹم کے ساتھ تعامل کرنے اور انٹرو ایکشن کے لیے کمانڈ لائن ٹولز کا ایک طاقتور سیٹ فراہم کرتا ہے۔ ان ٹولز کا سابقہ `ros2` ہے۔

*   `ros2 run <package_name> <executable_name>`: ROS 2 پیکج سے ایک ایگزیکوٹیبل چلائیں۔
*   `ros2 topic list`: فعال ٹاپکس کی فہرست دیں۔
*   `ros2 topic echo <topic_name>`: ٹاپک پر شائع شدہ پیغامات کو ظاہر کریں۔
*   `ros2 node list`: فعال نوڈس کی فہرست دیں۔
*   `ros2 service list`: فعال سروسز کی فہرست دیں۔
*   `ros2 param list`: نوڈ کے پیرامیٹر کی فہرست دیں۔
*   `ros2 interface show <msg/srv/action>`: میسج، سروس، یا ایکشن ٹائپ کی وضاحت دکھائیں۔

### بنیادی ROS 2 ورک سپیس اور پیکج کا سیٹ اپ
#### 1. ورک سپیس تخلیق کریں
ROS 2 ورک سپیس ایک ڈائریکٹری ہے جہاں آپ اپنے ROS 2 پیکجز کو ذخیرہ کرتے ہیں۔

```bash
mkdir -p ~/ros2_ws/src
cd ~/ros2_ws
```

#### 2. ورک سپیس کو شروع کریں اور تعمیر کریں

```bash
vcs import src < https://github.com/ros2/examples.git > --recursive
rosdep install --from-paths src --ignore-src -r --os=ubuntu:jammy
colcon build --packages-skip-build-flags "--skip-autodetect-src" --symlink-install
```

#### 3. سیٹ اپ فائلوں کو سورس کریں
اپنے ورک سپیس میں ROS 2 کمانڈز اور پیکجز کو استعمال کرنے کے لیے، آپ کو سیٹ اپ فائلوں کو سورس کرنا ہوگا۔

```bash
source /opt/ros/humble/setup.bash  # 'humble' کو اپنے ROS 2 ڈسٹری بیوشن کے ساتھ تبدیل کریں
source ~/ros2_ws/install/setup.bash
```

#### 4. نیا پیکج تخلیق کریں

```bash
cd ~/ros2_ws/src
ros2 pkg create --build-type ament_python my_robot_controller
```
یہ بنیادی پیکج کی ساخت کے ساتھ ایک ڈائریکٹری `my_robot_controller` تخلیق کرتا ہے، بشمول `package.xml` اور `setup.py`۔

## عملی مشقیں

### مشق 1: ROS 2 ماحولیاتی سیٹ اپ اور انٹرو ایکشن
**مقصد**: بنیادی ROS 2 ماحول سیٹ اپ کرنا اور کمانڈ لائن ٹولز کو چلتے ہوئے سسٹم کو دیکھنے کے لیے استعمال کرنا۔

**ہدایات**:
1.  یقینی بنائیں کہ آپ کے پاس ROS 2 (جیسے، ہمبلا یا آئرن) انسٹال ہے اور آپ کے سسٹم پر سورس ہے۔
2.  دو الگ الگ ٹرمنل ونڈوز کھولیں۔
3.  پہلے ٹرمنل میں، ROS 2 `talker` نوڈ چلائیں:
    ```bash
    ros2 run demo_nodes_cpp talker
    ```
4.  دوسرے ٹرمنل میں، اپنا ROS 2 ماحول سورس کریں۔
5.  `talker` نوڈ کو تلاش کرنے کے لیے `ros2 node list` استعمال کریں۔
6.  `talker` نوڈ کے ذریعہ شائع کردہ ٹاپک کو شناخت کرنے کے لیے `ros2 topic list` استعمال کریں۔
7.  شائع کردہ پیغامات دیکھنے کے لیے `ros2 topic echo <identified_topic_name>` استعمال کریں۔

**متوقع نتیجہ**:
*   پہلا ٹرمنل `[INFO] [talker]: Publishing: 'Hello World: <number>'` پیغامات دکھا رہا ہوگا۔
*   دوسرا ٹرمنل ٹاپک کی فہرست، نوڈ کی فہرست بشمول `talker`، اور ٹاپک سے "Hello World" پیغامات کو ظاہر کرے گا۔

### مشق 2: ایک کسٹم ROS 2 پیکج تخلیق کرنا اور تعمیر کرنا
**مقصد**: ایک کسٹم ROS 2 Python پیکج تخلیق کرنا اور اس کی تعمیر کی تصدیق کرنا۔

**ہدایات**:
1.  اپنے ROS 2 ورک سپیس `~/ros2_ws/src` پر جائیں۔
2.  `my_first_ros2_pkg` نام کا ایک نیا Python پیکج تخلیق کریں:
    ```bash
    ros2 pkg create --build-type ament_python my_first_ros2_pkg
    ```
3.  اپنے ورک سپیس کے روٹ پر واپس جائیں `~/ros2_ws`۔
4.  اپنا ورک سپیس تعمیر کریں (صرف آپ کا نیا پیکج تعمیر کیا جائے گا اگر کچھ اور تبدیل نہیں ہوا):
    ```bash
    colcon build
    ```
5.  اپنے ورک سپیس سیٹ اپ فائلوں کو سورس کریں (اگر پہلے سے سورس نہیں کیا گیا):
    ```bash
    source install/setup.bash
    ```
6.  تصدیق کریں کہ آپ کا پیکج پایا جا سکتا ہے:
    ```bash
    ros2 pkg prefix my_first_ros2_pkg
    ```

**متوقع نتیجہ**:
*   کامیاب `colcon build` آؤٹ پٹ، جس سے ظاہر ہو کہ `my_first_ros2_pkg` تعمیر ہو گیا تھا۔
*   `ros2 pkg prefix` کمانڈ آپ کے نئے پیکج کا انسٹالیشن پاتھ `~/ros2_ws/install/` کے اندر لوٹانا چاہیے۔

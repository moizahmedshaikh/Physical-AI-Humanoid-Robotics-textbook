# گزیبو سیمیولیشن ماحول

## تعلّم کے اہداف
*   روبوٹکس کی تحقیق اور ترقی میں گزیبو کے کردار کی وضاحت کریں۔
*   گزیبو سیمیولیشن آرکیٹیکچر کے کلیدی اجزاء کی شناخت کریں اور ان کی وضاحت کریں۔
*   گزیبو صارف انٹرفیس لانچ کریں اور نیویگیٹ کریں اور سیمیولیٹڈ دنیا سے تعامل کریں۔
*   گزیبو کے اندر موجود روبوٹ ماڈلز اور ماحول کو لوڈ کریں اور مینوپولیٹ کریں۔
*   گزیبو دنیا اور ماڈلز کی وضاحت کے لیے SDF (سیمیولیشن تفصیل کا فارمیٹ) کے تصور کو سمجھیں۔

## جامع مواد

گزیبو ایک طاقتور 3D روبوٹکس سیمیولیٹر ہے جو ڈیولپرز کو الگورتھم کی ایک درست اور کارآمد طریقے سے جانچ کرنے، روبوٹس کی ڈیزائن کرنے، اور ورچوئل ماحول میں تربیت کرنے کی اجازت دیتا ہے۔ یہ روبوٹس کی ایک آبادی کو سیمیولیٹ کرنے، پیچیدہ اندر اور باہر کے ماحول، اور حقیقی سینسر ڈیٹا پیدا کرنے کی صلاحیت فراہم کرتا ہے۔ جسمانی مصنوعی ذہانت اور ہیومنوائڈ روبوٹکس کے لیے، گزیبو نظریاتی تصورات اور حقیقی دنیا کے اتارنے کے درمیان خلا کو پُر کرنے کے لیے ایک لازمی ٹول ہے۔

### روبوٹکس میں گزیبو کا کردار

گزیبو کا بنیادی کردار ہائی فائیڈلٹی فزکس سیمیولیشن ماحول فراہم کرنا ہے۔ یہ روبوٹ انجینئرز اور AI محققین کو اجازت دیتا ہے:
*   **الگورتھم کی جانچ**: فزیکل ہارڈ ویئر کی ضرورت کے بغیر کنٹرول، نیویگیشن، اور ادراک الگورتھم تیار کریں اور ڈیبگ کریں۔
*   **ترقی کو تیز کریں**: فزیکل پروٹو ٹائپس کی حدود اور اخراجات سے بچ کر روبوٹ ڈیزائنز اور سافٹ ویئر پر تیزی سے دوبارہ کام کریں۔
*   **حفاظت**: محفوظ، ورچوئل سیٹنگ میں خطرناک منظار یا ناکامی کے طریقے کی جانچ کریں۔
*   **دوبارہ تعمیل کرنا**: مسلسل ابتدائی حالت کے ساتھ تجربات چلائیں، نتائج کے قابل اعتماد موازنے کو فعال کریں۔
*   **ڈیٹا جنریشن**: مشین لرننگ ماڈلز کی تربیت کے لیے سیمیولیٹڈ سینسر ڈیٹا کے بڑے ڈیٹا سیٹس تیار کریں (جیسے، چیز کی پہچان، SLAM کے لیے)۔

### گزیبو آرکیٹیکچر کے اجزاء

گزیبو کو چند کلیدی اجزاء کے گرد تعمیر کیا گیا ہے جو سیمیولیشن ماحول کو بنانے کے لیے مل کر کام کرتے ہیں:

1.  **فزکس انجن**: اس کے مرکز میں، گزیبو مختلف ہائی پرفارمنس فزکس انجن (جیسے، ODE، بُلیٹ، DART، سِم بอดی) کے ساتھ ضم ہوتا ہے۔ یہ انجن قوتیں، کالیژن، اور ڈائنیمکس کا حساب لگاتے ہیں تاکہ روبوٹ کی حرکت اور اشیاء کے ساتھ تعامل کو حقیقت پسندانہ طور پر سیمیولیٹ کیا جا سکے۔
2.  **رینڈرنگ انجن**: سیمیولیشن کی بصری نمائندگی فراہم کرتا ہے، جیسے اوگر3D لائبریریز کا استعمال کرتے ہوئے۔ یہ 3D گریفکس پیدا کرتا ہے جو آپ گزیبو GUI میں دیکھتے ہیں، بشمول حقیقی لائٹنگ، سایہ، اور ٹیکسچر۔
3.  **سینسر ماڈلز**: گزیبو عام روبوٹ سینسرز (کیمرے، لائیڈار، آئی ایم یو، فورس/ٹورک سینسرز وغیرہ) کے لیے ماڈلز فراہم کرتا ہے۔ یہ ماڈلز اسی طرح کا سینسر ڈیٹا سیمیولیٹ کرتے ہیں جو ایک فزیکل روبوٹ ادراک کرے گا، اکثر حقیقی دنیا کی حالت کی نقل کرنے کے لیے قابل ترتیب نوائس اور نقصانات شامل کرتے ہیں۔
4.  **دنیا اور ماڈل فائلیں (SDF)**: گزیبو کے اندر ماحول اور روبوٹس کو SDF (سیمیولیشن تفصیل کا فارمیٹ) فائلز کا استعمال کرتے ہوئے تفصیل دی جاتی ہے۔ SDF ایک جامع XML فارمیٹ ہے جو چیز کی نمائش، گراؤنڈ پلینز سے لے کر تفصیلی روبوٹ کائینمیٹکس، ڈائنیمکس، اور سینسر کنفیگریشنز تک کی وضاحت کرتا ہے۔
5.  **پلگ انز**: گزیبو کی قابلیت کو C++ پلگ انز کے ذریعے وسعت دی جا سکتی ہے۔ یہ صارفین کو بیرونی لائبریریز (جیسے ROS 2) کے ساتھ انٹرفیس کرنے، کسٹم سینسرز شامل کرنے، روبوٹ جوائینٹس کنٹرول کرنے، یا رن ٹائم کے دوران فزکس پیرامیٹر میں ترمیم کرنے کی اجازت دیتے ہیں۔
6.  **گریفکل صارف انٹرفیس (GUI)**: ویژولائز، کنٹرول، اور سیمیولیشن کو ڈیبگ کرنے کے لیے ایک صارف دوست انٹرفیس۔ یہ صارفین کو سیمیولیشن کو روکنے/چلانے، ری سیٹ کرنے، اشیاء کو مینوپولیٹ کرنے، اور سینسر آؤٹ پٹس کو انسپیکٹ کرنے کی اجازت دیتا ہے۔

### گزیبو کو لانچ کرنا اور نیویگیٹ کرنا

#### گزیبو کو لانچ کرنا
گزیبو کو مخصوص دنیا کی فائل کے ساتھ یا بغیر لانچ کیا جا سکتا ہے۔

```bash
# گزیبو خالی دنیا GUI لانچ کریں
ros2 launch gazebo_ros gazebo.launch.py

# مخصوص دنیا (جیسے، خالی دنیا) کے ساتھ گزیبو لانچ کریں
ros2 launch gazebo_ros gazebo.launch.py gazebo_args:="-r worlds/empty.world"

# مخصوص دنیا (جیسے، عام گارڈن دنیا) کے ساتھ گزیبو لانچ کریں
ros2 launch gazebo_ros gazebo.launch.py gazebo_args:="-r worlds/garden.world"
```

#### GUI میں نیویگیٹ کرنا
ایک بار جب گزیبو لانچ ہو جاتا ہے، تو آپ 3D ماحول کے ساتھ تعامل کر سکتے ہیں:
*   **ماؤس کنٹرولز**: کیمرہ کو اوربٹ، پین، اور زوم کرنے کے لیے ماؤس استعمال کریں۔
    *   **بائیں کلک + ڈریگ**: چکر/آربٹ ویو۔
    *   **دائیں کلک + ڈریگ**: پین ویو۔
    *   **سکرول ویل**: زوم ان/آؤٹ۔
*   **ٹول بار**: سیمیولیشن کو روکنے/چلانے، دنیا کو ری سیٹ کرنے، سادہ شکلیں (بکسز، سپیئرز) شامل کرنے، اور ماڈلز منتخب کرنے کے لیے کنٹرولز پر مشتمل ہے۔
*   **دنیا ٹری**: بائیں طرف واقع، سیمیولیشن میں تمام اجزاء (ماڈلز، لائٹس، سینسرز) دکھاتا ہے۔ آپ ان کی خصوصیات کو انسپیکٹ یا تبدیل کرنے کے لیے یہاں سے اجزاء کو منتخب کر سکتے ہیں۔
*   **انسیٹ پینل**: دنیا میں عام ماڈلز (جیسے، سادہ شکلیں، گزیبو ماڈل ڈیٹا بیس سے روبوٹس) کو داخل کرنے کی اجازت دیتا ہے۔

### ماڈلز کو لوڈ کرنا اور مینوپولیٹ کرنا

گزیبو ماڈلز عام طور پر SDF فائلز میں یا URDF فائلز سے تبدیل کیے جاتے ہیں (جس میں اکثر گزیبو مخصوص ایکسٹینشنز شامل ہوتے ہیں، کبھی کبھی XACRO کے ذریعے)۔

**ماڈلز کو لوڈ کرنا**:
ماڈلز کو لانچ فائل کے ذریعے اسٹارٹ اپ پر یا GUI کے ذریعے متحرک طور پر داخل کیا جا سکتا ہے۔ `spawn_entity.py` ٹول کو ROS 2 سے URDF/SDF ماڈلز کو اسپون کرنے کے لیے عام طور پر استعمال کیا جاتا ہے۔

```bash
# مثال: ایک سادہ ماڈل اسپون کرنا (ایک پیکج کی ضرورت ہوتی ہے جس میں ماڈل ہو، مثلاً turtlebot3_gazebo)
# پہلے اپنا ROS 2 ماحول اور turtlebot3_gazebo پیکج سورس کریں
# ros2 launch turtlebot3_gazebo turtlebot3_world.launch.py # یہ ٹرٹل بوٹ3 کے ساتھ گزیبو لانچ کرتا ہے

# کسی مخصوص ماڈل کو دستی طور پر اسپون کرنے کے لیے (اگر آپ کے پاس ایک SDF/URDF فائل ہو)
# ros2 run gazebo_ros spawn_entity.py -entity my_robot -file /path/to/my_robot.urdf -x 0 -y 0 -z 0
```

**GUI میں ماڈلز کو مینوپولیٹ کرنا**:
1.  **ایک ماڈل منتخب کریں**: 3D ویو یا دنیا ٹری میں کسی ماڈل پر کلک کریں۔
2.  **ٹرانسلیشن/ریوٹیشن**: منتخب کردہ ماڈل کو منتقل یا اس کا رخ کرنے کے لیے ٹول بار ویجیٹس (ٹرانسلیٹ، ریوٹیٹ) استعمال کریں۔ آپ براہ راست ہینڈلز کو ڈریگ بھی کر سکتے ہیں۔
3.  **خصوصیات**: بائیں پینل میں، آپ منتخب کردہ ماڈل کی جسمانی خصوصیات، پوز، اور دیگر صفات کو تبدیل کر سکتے ہیں۔

### سیمیولیشن تفصیل کا فارمیٹ (SDF)

SDF XML فارمیٹ ہے جسے گزیبو ماڈلز سے لے کر مکمل سیمیولیشن دنیا تک کی ہر چیز کی وضاحت کے لیے استعمال کیا جاتا ہے۔ یہ URDF کے مقابلے میں فزکس اور سینسر کی خصوصیات کے لحاظ سے زیادہ اظہار کے قابل ہے۔

**کلیدی SDF عناصر**:
*   **`<world>`**: دنیا کی فائل کا روٹ عنصر، جس میں لائٹس، ماڈلز، زمین، فزکس ترتیبات شامل ہیں۔
*   **`<model>`**: لنکس، جوائینٹس، اور پلگ انز کا مجموعہ جو دنیا میں ایک روبوٹ یا ایک چیز کی وضاحت کرتا ہے۔
*   **`<link>`**: URDF کی طرح، ایک سخت جسم کی وضاحت کرتا ہے جس میں ویژول، کالیژن، اور ادراک کی خصوصیات ہیں۔
*   **`<joint>`**: URDF کی طرح، لنکس کے درمیان کنکشن کی وضاحت کرتا ہے۔
*   **`<include>`**: دیگر SDF ماڈل فائلز کو شامل کر کے ماڈولریٹی کی اجازت دیتا ہے۔
*   **`<plugin>`**: گزیبو پلگ انز کو ماڈلز یا دنیا سے منسلک کرتا ہے۔
*   **`<light>`**: لائٹ ماخذ کی وضاحت کرتا ہے۔
*   **`<gui>`**: گزیبو GUI کی کنفیگریشن۔

**مثال: سادہ SDF دنیا (`empty.world`)**
```xml
<?xml version="1.0" ?>
<sdf version="1.7">
  <world name="empty_world">
    <include>
      <uri>model://sun</uri>
    </include>
    <include>
      <uri>model://ground_plane</uri>
    </include>
  </world>
</sdf>
```

## عملی مشقیں

### مشق 1: گزیبو کو لانچ کرنا اور ایک ماڈل داخل کرنا
**مقصد**: ایک خالی گزیبو دنیا کو لانچ کرنا اور متحرک طور پر ایک بنیادی ماڈل داخل کرنا۔

**ہدایات**:
1.  ایک ٹرمنل کھولیں اور GUI کے ساتھ ایک خالی گزیبو دنیا لانچ کریں:
    ```bash
    ros2 launch gazebo_ros gazebo.launch.py
    ```
2.  جب گزیبو لوڈ ہو جائے، تو بائیں پینل میں "انسیٹ" ٹیب میں، ایک سادہ ماڈل تلاش کریں اور داخل کریں (جیسے، "سادہ سپیئرز" یا "کیوب")۔
3.  گزیبو GUI میں ٹرانسلیٹ اور ریوٹیٹ ٹولز کا استعمال کر کے داخل کردہ چیز کو منتقل کریں اور اس کا رخ کریں۔
4.  سیمیولیشن کو روکیں اور پھر چلائیں تاکہ فزکس کے اثر کا مشاہدہ کیا جا سکے (جیسے، اگر آپ نے چیز کو اونچائی سے گرایا ہو)۔

**متوقع نتیجہ**:
*   ایک خالی گزیبو دنیا ظاہر ہو گی۔
*   3D ماحول کے اندر داخل کی گئی اور قابلِ مینوپولیٹ ایک سادہ چیز۔
*   بنیادی فزکس کا مشاہدہ (جیسے، چیز گرنے، گھومنا)۔

### مشق 2: SDF سٹرکچر کو سمجھنا
**مقصد**: ایک سادہ SDF ماڈل فائل کا معائنہ کرنا اور اس کی سٹرکچر کو سمجھنا۔

**ہدایات**:
1.  اپنے سسٹم پر ایک سادہ SDF ماڈل فائل تلاش کریں۔ ایک اچھی مثال گزیبو ماڈل ڈیٹا بیس سے `ground_plane.sdf` یا `sun.sdf` ہو سکتی ہے۔ یہ عام طور پر `/usr/share/gazebo-X/models/` میں پائے جاتے ہیں (X کو اپنے گزیبو ورژن کے ساتھ تبدیل کریں، مثلاً `gazebo-11`)۔
2.  ایک ٹیکسٹ ایڈیٹر کا استعمال کرتے ہوئے SDF فائل کھولیں۔
3.  روٹ `<sdf>` ٹیگ کو تلاش کریں اور پھر اہم `<model>` (یا `<world>`) ٹیگ۔
4.  `<link>` عناصر اور ان کے `<visual>`، `<collision>`، اور `<inertial>` ذیلی عناصر تلاش کریں اگر موجود ہوں۔
5.  کوئی `<joint>` عناصر یا `<include>` ہدایات کی شناخت کریں۔

**متوقع نتیجہ**:
SDF فائل میں آپ کی طرف سے شناخت کردہ کلیدی اجزاء کا ایک مختصر لکھا ہوا خلاصہ (1-2 پیراگراف) جس میں کم از کم ایک `<link>`، `<visual>`، اور `<geometry>` ٹیگ، اور کوئی `<include>` ٹیگس کا تذکرہ ہو۔

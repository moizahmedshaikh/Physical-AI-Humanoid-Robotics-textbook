# روبوٹکس سیمیولیشن کے لیے یونیٹی انضمام

## تعلّم کے اہداف
*   یونیٹی کو ROS 2 جیسے روبوٹکس فریم ورکس کے ساتھ ضم کرنے کے فوائد اور استعمال کے معاملات کو سمجھیں۔
*   جسمانی مصنوعی ذہانت کے لیے حقیقی سیمیولیشن اور ویژولائزیشن پلیٹ فارم کے طور پر یونیٹی کے کردار کی وضاحت کریں۔
*   یونیٹی کو روبوٹکس ڈیولپمنٹ کے لیے سیٹ اپ کریں، بشمول ضروری پیکجز اور کنفیگریشنز۔
*   روبوٹ کنٹرول اور سینسر ڈیٹا کے تبادلے کے لیے یونیٹی اور ROS 2 کے درمیان بنیادی مواصلت نافذ کریں۔
*   دیگر مخصوص روبوٹکس سیمیولیٹرز کے مقابلے میں سیمیولیشن کے لیے یونیٹی کو کب منتخب کرنا ہے کا جائزہ لیں۔

## جامع مواد

جیسے کہ گزیبو ایک طاقتور فزکس بیسڈ سیمیولیٹر ہے، یونیٹی، ایک نمایاں ریل ٹائم 3D ڈیولپمنٹ پلیٹ فارم، انتہائی حقیقی اور تعاملی ماحول تخلیق کرنے کے لیے بے مثال گریفکل فائیڈلٹی، اعلی درجے کی رینڈرنگ کی صلاحیتیں، اور ایک غنی ماحول فراہم کرتا ہے۔ ROS 2 جیسے روبوٹکس فریم ورکس کے ساتھ یونیٹی کو ضم کرنا جسمانی مصنوعی ذہانت کے لیے نئی ممکنات کھول دیتا ہے، خاص طور پر ان کاموں کے لیے جن میں تفصیلی بصری ادراک، انسان-روبوٹ تعامل، یا فوٹو ریل سیمیولیشن کی ضرورت ہو۔

### روبوٹکس سیمیولیشن کے لیے یونیٹی کے فوائد

یونیٹی روبوٹکس سیمیولیشن کے شعبے میں کئی منفرد فوائد لاتا ہے:
*   **اعلی بصری فائیڈلٹی**: بہترین گریفکس، لائٹنگ، اور رینڈرنگ کی صلاحیتیں انتہائی حقیقی ماحول تخلیق کرتی ہیں، جو وژن بیسڈ AI ماڈلز کی تربیت اور انسان-روبوٹ تعامل کے مطالعات کے لیے اہم ہے۔
*   **غنی اثاثہ اسٹور**: 3D ماڈلز، ٹیکسچرز، اور ماحول کی وسیع لائبریری تک رسائی، دنیا کی تخلیق کو تیز کرتا ہے۔
*   **تعاملی ماحول**: متحرک اور تعاملی مناظر کو آسانی سے تخلیق کرنا، پیچیدہ تجرباتی ترتیبات کی اجازت دیتا ہے۔
*   **کراس پلیٹ فارم ڈیپلائمنٹ**: ڈیسک ٹاپ، VR/AR، اور ویب سمیت مختلف پلیٹ فارم پر ڈیپلائمنٹ کو سپورٹ کرتا ہے۔
*   **گیم ڈیولپمنٹ ماحول**: ایک پختہ ڈیولپمنٹ ماحول کا فائدہ اٹھاتا ہے جس میں طاقتور اسکرپٹنگ (C#)، ڈیبگنگ ٹولز، اور ایک بڑی کمیونٹی ہے۔
*   **ادراک ڈیٹا سیٹ جنریشن**: مشین لرننگ کے لیے مصنوعی ڈیٹا سیٹس تخلیق کرنے کے لیے مثالی، ماحول، لائٹنگ، اور چیز کی خصوصیات پر مکمل کنٹرول کے ساتھ۔

### جسمانی مصنوعی ذہانت میں یونیٹی کے استعمال کے معاملات

*   **ریانفورسمنٹ لرننگ (RL)**: پیچیدہ، بصری طور پر امیر ماحول میں AI ایجنٹس کی تربیت جہاں حقیقی بصری ان پٹ اہم ہو۔
*   **انسان-روبوٹ تعامل (HRI)**: سماجی تعاملات، اظہار والے روبوٹ کے رویے، اور صارف انٹرفیسز کی نقل کرنا۔
*   **ٹیلی آپریشن**: روبوٹس کے دور دراز کنٹرول کے لیے غوطہ خیز اور حقیقی ورچوئل ماحول فراہم کرنا۔
*   **پاتھ پلاننگ اور نیویگیشن**: تفصیلی اندر/باہر کے مناظر میں نیویگیشن الگورتھم کی جانچ کرنا۔
*   **ڈیجیٹل ٹوئنز**: مانیٹرنگ اور پیش گوئی کی مرمت کے لیے جسمانی روبوٹس اور ماحول کے انتہائی درست ورچوئل نقل تخلیق کرنا۔
*   **تدریس اور تربیت**: روبوٹکس کے لیے تعاملی تدریسی اوزار تیار کرنا۔

### یونیٹی کو روبوٹکس ڈیولپمنٹ کے لیے سیٹ اپ کرنا

ROS 2 کے ساتھ یونیٹی کو ضم کرنے کے لیے، آپ عام طور پر `Unity-Technologies/ROS-TCP-Connector` گیٹ ہب ریپوزیٹری سے `ROS-TCP-Endpoint` اور `ROS-TCP-Connector` جیسے پیکجز استعمال کرتے ہیں۔ یہ پیکجز TCP ساکٹس کے ذریعے یونیٹی اور ROS 2 کے درمیان کم تاخیر والی مواصلت کو فعال کرتے ہیں۔

**سیٹ اپ کے لیے کلیدی اقدامات**:
1.  **یونیٹی انسٹال کریں**: یونیٹی ہب اور ایک مناسب یونیٹی ایڈیٹر ورژن (مثلاً LTS ورژن) ڈاؤن لوڈ اور انسٹال کریں۔
2.  **ایک نیا یونیٹی پروجیکٹ تخلیق کریں**: ایک نیا 3D (URP یا HDRP اعلی فائیڈلٹی کے لیے) پروجیکٹ شروع کریں۔
3.  **ROS-TCP-Connector درآمد کریں**: `ROS-TCP-Connector` یونیٹی پیکج ڈاؤن لوڈ کریں یا اسے پیکج مینیجر کے ذریعے درآمد کریں (اگر OpenUPM کے ذریعے دستیاب ہو)۔
    *   `ROS-TCP-Endpoint`: ایک ROS 2 نوڈ جو ROS کی طرف چلتا ہے، TCP کنکشنز کے لیے ایک سرور کے طور پر کام کرتا ہے۔
    *   `ROS-TCP-Connector`: یونیٹی کلائنٹ سائیڈ لائبریریز جو `ROS-TCP-Endpoint` سے منسلک ہوتی ہیں اور میسج سیریلائزیشن/ڈی سیریلائزیشن کو سنبھالتی ہیں۔
4.  **`ROS-TCP-Endpoint` کی ترتیب دیں**: ROS 2 کی طرف، `ros_tcp_endpoint` پیکج کو تعمیر کریں اور چلائیں۔
    ```bash
    # اپنے ROS 2 ورک سپیس میں، ریپو کو کلون کرنے کے بعد:
    cd ~/ros2_ws/src/ROS-TCP-Connector/ros_tcp_endpoint
    colcon build
    source install/setup.bash
    ros2 run ros_tcp_endpoint endpoint_node
    ```
5.  **یونیٹی `RosConnection` کی ترتیب دیں**: اپنے یونیٹی پروجیکٹ میں، ایک گیم آبجیکٹ میں `RosConnection` کمپوننٹ شامل کریں۔ `Ros IP` (اپنے ROS 2 مشین کا IP ایڈریس) اور `Port` (ڈیفالٹ 10000) سیٹ کریں۔

### یونیٹی اور ROS 2 کے درمیان مواصلت

`ROS-TCP-Connector` یونیٹی کے اندر C# اسکرپٹس فراہم کرتا ہے تاکہ ROS 2 میسج، سروسز، اور ایکشنز بھیجے اور وصول کیے جا سکیں۔

#### 1. یونیٹی سے ROS 2 میسج شائع کرنا

یونیٹی ROS 2 ٹاپکس پر سیمیولیٹڈ سینسر ڈیٹا (جیسے، کیمرہ امیجز، لائیڈار پوائنٹ کلاؤڈز) یا روبوٹ سٹیٹس شائع کر سکتا ہے۔

**مثال: یونیٹی سے ایک سٹرنگ میسج شائع کرنا (C#)**
```csharp
using UnityEngine;
using Unity.Robotics.ROSTCPConnector;
using RosMessageTypes.Std;

public class UnityPublisher : MonoBehaviour
{
    RosConnection ros;
    public string topicName = "/unity_chatter";
    private StringMsg stringMessage;

    void Start()
    {
        ros = RosConnection.Get                  Instance();
        ros.RegisterPublisher<StringMsg>(topicName);
        stringMessage = new StringMsg();
    }

    void Update()
    {
        stringMessage.data = "Hello from Unity! Time: " + Time.time;
        ros.Publish(topicName, stringMessage);
    }
}
```

#### 2. یونیٹی میں ROS 2 میسج کو سبسکرائب کرنا

یونیٹی کنٹرول کمانڈز (جیسے، روبوٹ حرکت کے لیے `cmd_vel`) یا ROS 2 سے فیڈ بیک کو سبسکرائب کر سکتا ہے۔

**مثال: یونیٹی میں ایک سٹرنگ میسج کو سبسکرائب کرنا (C#)**
```csharp
using UnityEngine;
using Unity.Robotics.ROSTCPConnector;
using RosMessageTypes.Std;

public class UnitySubscriber : MonoBehaviour
{
    public string topicName = "/ros_chatter";

    void Start()
    {
        RosConnection.Get                  Instance().Subscribe<StringMsg>(topicName, RosStringCallback);
    }

    void RosStringCallback(StringMsg stringMessage)
    {
        Debug.Log("Received from ROS: " + stringMessage.data);
    }
}
```

#### 3. سروسز اور ایکشنز

`ROS-TCP-Connector` ROS 2 سروسز اور ایکشنز کی بھی حمایت کرتا ہے، زیادہ پیچیدہ، دو طرفہ تعاملات کی اجازت دیتا ہے۔ یہ یونیٹی سے ROS 2 کنٹرول اسٹیک کو مخصوص کمانڈز (سروسز) یا طویل وقت تک جاری رہنے والے روبوٹ کاموں (ایکشنز) کو بھیجنے کے لیے اہم ہے۔

### سیمیولیشن کے لیے یونیٹی کب منتخب کریں

جیسے کہ گزیبو اعلی درجے کی فزکس میں بہتر ہے اور ROS 2 کے نیٹیو ماحول کے ساتھ گہرائی سے ضم ہے، یونیٹی ایک مضبوط انتخاب بن جاتا ہے جب:
*   **فوٹو ریلزم سب سے اہم ہو**: وژن AI تربیت کے لیے جہاں امیج کی کوالیٹی، لائٹنگ، اور ٹیکسچرز اہم ہوں۔
*   **پیچیدہ انسان-روبوٹ تعامل (HRI)**: جب حقیقی انسانی اوتار، چہرے کے اظہار، اور نازک سماجی اشارے کی ضرورت ہو۔
*   **کسٹم سینسر ماڈلنگ**: جب گریفکل درستگی کے ساتھ انتہائی مخصوص یا غیر معیاری سینسر رویوں کو سیمیولیٹ کیا جانا ضروری ہو۔
*   **صارف تجربہ (UX) اور ویژولائزیشن**: دلچسپ مظاہر، ٹیلی آپریشن انٹرفیسز، یا تدریسی اوزار کے لیے۔
*   **ڈیٹا سیٹ جنریشن**: مشین لرننگ ماڈلز (جیسے، چیز کا پوز ایسٹیمیشن، سیمینٹک سیگمینٹیشن) کے لیے وسیع، متنوع، اور اینوٹیٹڈ مصنوعی ڈیٹا سیٹس تخلیق کرنا۔
*   **غیر روبوٹکس سافٹ ویئر کے ساتھ گہرائی سے انضمام**: جب روبوٹکس کو دیگر یونیٹی مخصوص خصوصیات یا تیسرے فریق کے اثاثوں کے ساتھ جوڑا جائے۔

تاہم، یہ اہم ہے کہ غور کیا جائے کہ یونیٹی کا فزکس انجن ہمیشہ کچھ انتہائی حساس روبوٹکس تحقیق کے لیے ضروری عددی درستگی سے مماثل نہیں ہو سکتا، اور ROS 2 میسج کو ایڈاپٹ کرنا یا نئے سینسر کی اقسام کو ضم کرنا گزیبو کی نیٹیو حمایت کے مقابلے میں زیادہ دستی سیٹ اپ کی ضرورت ہو سکتی ہے۔

## عملی مشقیں

### مشق 1: ROS-TCP-Endpoint سیٹ اپ کریں اور کنیکٹویٹی کی جانچ کریں
**مقصد**: ROS 2 اور یونیٹی کے درمیان ایک بنیادی مواصلت کا لنک قائم کرنا۔

**ہدایات**:
1.  **ROS 2 سائیڈ**: `Unity-Technologies/ROS-TCP-Connector` ریپوزیٹری کو اپنے ROS 2 ورک سپیس `src` ڈائریکٹری میں کلون کریں۔ `ros_tcp_endpoint` پیکج کو تعمیر کریں اور `endpoint_node` چلائیں:
    ```bash
    cd ~/ros2_ws/src
    git clone https://github.com/Unity-Technologies/ROS-TCP-Connector.git
    cd ~/ros2_ws
    colcon build --packages-select ros_tcp_endpoint
    source install/setup.bash
    ros2 run ros_tcp_endpoint endpoint_node
    ```
2.  **یونیٹی سائیڈ**: ایک نیا یونیٹی پروجیکٹ تخلیق کریں۔ کلون کردہ ریپوزیٹری سے `com.unity.robotics.rostcpconnector` پیکج درآمد کریں۔ ایک خالی گیم آبجیکٹ تخلیق کریں، ایک `RosConnection` کمپوننٹ شامل کریں، اور اس کا `Ros IP` اپنے ROS 2 مشین کے IP ایڈریس پر اور `Port` 10000 پر ترتیب دیں۔ یونیٹی منظر چلائیں۔
3.  **کنکشن کی تصدیق کریں**: یونیٹی اور ROS 2 ٹرمنل دونوں میں کنسول لاگز چیک کریں تاکہ کامیاب کنکشن کی نشاندہی والے پیغامات دکھائی دیں۔

**متوقع نتیجہ**:
*   ROS 2 ٹرمنل جس میں "ROS-TCP-Endpoint initialized, listening on port 10000" جیسے پیغامات دکھائی دیں۔
*   یونیٹی کنسول جس میں "ROS TCP Connection established." جیسے پیغامات دکھائی دیں۔

### مشق 2: ROS 2 کے لیے ایک سادہ یونیٹی پبلشر نافذ کریں
**مقصد**: یونیٹی سے ROS 2 ٹاپک پر ایک کسٹم سٹرنگ میسج شائع کرنا اور اس کی وصولی کی تصدیق کرنا۔

**ہدایات**:
1.  مشق 1 سے جاری رکھیں جس میں ROS 2 - یونیٹی کنکشن قائم ہو چکا ہو۔
2.  **یونیٹی سائیڈ**: یونیٹی میں ایک نیا C# اسکرپٹ (مثلاً `UnityStringPublisher.cs`) تخلیق کریں۔ اوپر "یونیٹی سے ROS 2 میسج شائع کرنا" سیکشن میں فراہم کردہ کوڈ شامل کریں۔ اس اسکرپٹ کو اپنے منظر میں ایک خالی گیم آبجیکٹ سے منسلک کریں۔ یقینی بنائیں کہ `topicName` سیٹ کر دیا گیا ہو (مثلاً، `/unity_chatter`)۔
3.  **ROS 2 سائیڈ**: ایک نئے ٹرمنل میں (ROS 2 اور آپ کے ورک سپیس سیٹ اپ کو سورس کرنے کے بعد)، `ros2 topic echo /unity_chatter` چلائیں۔
4.  **چلائیں**: یونیٹی منظر چلائیں اور ROS 2 ٹرمنل کو دیکھیں۔

**متوقع نتیجہ**:
*   یونیٹی کنسول جس میں شائع کرنے کے بارے میں ڈیبگ پیغامات دکھائی دیں۔
*   ROS 2 ٹرمنل جس میں "/unity_chatter" ٹاپک سے ایکو ہونے والے "Hello from Unity! Time: ..." پیغامات دکھائی دیں۔
